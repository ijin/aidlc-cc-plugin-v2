# Working in this repo (agent guide)

This repo packages the **v2** rewrite of [awslabs/aidlc-workflows](https://github.com/awslabs/aidlc-workflows)
as a Claude Code plugin (`aidlc-v2`). Read `README.md` for the full picture; this file is the short
list of **rules and invariants** — break one and you break the plugin or its provenance.

This file is **committed and public** — keep it environment-neutral (no machine paths, tooling, or
provider names). Any *local* workflow notes (how a particular machine runs the review/smoke) go in
`CLAUDE.local.md`, which is gitignored.

## Layout & the build model (internalize this first)

- `src/` is a **pristine, vendored mirror** of upstream's `src/` at the commit pinned in
  `UPSTREAM.lock`. **NEVER hand-edit `src/`.** It is replaced wholesale by `sync-upstream.sh`; a
  manual edit is detected and rejected (the skew guard / dist-freshness check).
- `dist/claude/` is the **built, committed plugin** — the *only* thing that ships
  (marketplace `source: ./dist/claude`). It is generated by `targets/claude/build.mjs`. **NEVER
  hand-edit `dist/`** — rebuild instead. `src/`, `targets/`, `test/`, `.claude/`, `.design/` do NOT ship.
- All Claude-specific adaptation lives in **`targets/claude/`** (the build, hooks, sync/triage/score/
  tag scripts). That is where changes belong — not in `src/`.
- After any change that feeds the build, **rebuild and commit `dist/` in the same change**
  (`node targets/claude/build.mjs && git add dist/`). A pre-commit hook + CI enforce that committed
  `dist/` equals a fresh build; `git config core.hooksPath .githooks` to enable the hook locally.

## Syncing from upstream

- To adopt a new upstream snapshot, use **`./targets/claude/sync-upstream.sh <sha>`** (or the
  guided `release-upstream` skill). It is the only correct path: it sparse-checkouts upstream `src/`,
  rebuilds, and rewrites the lock on a clean build — and never auto-commits.
- **Do NOT run `git subtree pull`.** `src/` is no longer a subtree; that command would map upstream's
  repo root into `src/` and corrupt it.
- Upstream v2 is a **tagless, force-pushable dev branch**, so a human reviews every snapshot. The
  decision to adopt is human; only the mechanics are automated.

## Verification (run before claiming done)

- `npm test` runs the free gates: contract-drift meta-tests, T1 triage classifier, T3 scorer, and the
  `dist/`-freshness guard. **Always green before committing.**
- **T2 (behavioral smoke) is OFF by default** — it makes billable LLM/API calls and has found no
  plugin defects in practice. Run it only when warranted: `sync-triage.mjs` emits `t2b.advised` (true
  only when the diff touches the runtime behavioral surface). Don't run T2 on content-only changes.
- **If the build fails on a contract check**, it names what upstream changed; the fix is in
  `targets/claude/` (e.g. unmapped agent tool → `TOOL_MAP`; un-rewritten Kiro primitive in `dist/` →
  `kiroToClaude()`; unexpected `src/` entry → `REQUIRED_SRC_DIRS` + build steps; agent-schema change →
  `KNOWN_AGENT_KEYS`/`buildAgents()`). Fix the adapter, never `src/` or `dist/`. Full table:
  `MAINTAINERS.md`.

## How to make changes here

- **Independent review is part of "done."** For any substantive change (build/adapter logic, the
  contract, the verification tiers, the sync flow): make the change → get an independent adversarial
  review → reach explicit agreement → *then* it's done. Do not treat "committed" as "finished" before
  the review loop closes.
- Match upstream's algorithms **faithfully** when porting (e.g. the T3 scorer is parity-locked to
  upstream's Python — verify against it, don't approximate).
- Validate YAML with `actionlint`; no trailing whitespace on blank lines.

## Public-repo hygiene

- This repo is intended to go public. Commit messages are **plugin/snapshot-focused, for an outside
  reader** — do NOT use internal process/tooling framing in public commit messages. The
  `release-upstream` skill already produces clean, snapshot-focused text.
- `src/` is vendored from upstream (MIT-0, Copyright Amazon); keep the README's License & attribution
  section accurate. This is an independent community port, not affiliated with AWS.
